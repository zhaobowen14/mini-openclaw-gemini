# スライドの条件
良いスライドとは「読ませずに理解させる」もの。
## ① 1スライド＝1メッセージ
「結局何が言いたいのか」が一瞬で分かることが重要。  
複数テーマを詰め込まず、1枚につき1つの主張に絞る。
## ② タイトルが結論になっている
タイトルだけで内容が理解できる状態が理想。  
「サービスの特徴」ではなく、「当社サービスは低コストかつ短納期を実現」とする。
## ③ 文字量は最小限（3〜5行）
読み込ませるのではなく、見て理解できる量にする。  
目安は1行20〜30文字 × 3〜5行。
## ④ 箇条書きは構造を揃える
並列関係や因果関係を明確にする。  
文末や表現形式を揃えることで、理解しやすくなる。
## ⑤ 強調は最小限にする
太字や色の使いすぎは逆効果。  
1スライドあたり1〜2箇所に絞る。
## ⑥ 図解・ビジュアルを活用する
文章だけで説明せず、図やグラフを使う。  
視覚情報の方が理解速度が速い。
## ⑦ 余白を十分に確保する
情報を詰め込みすぎない。  
スライドの30〜50%は空白でも問題ない。
## ⑧ フォントとサイズを統一する
タイトル・本文・補足でサイズや役割を統一する。  
見た目の整合性が信頼感につながる。
## ⑨ 日本語は簡潔にする
冗長な表現を避け、短く言い切る。  
「〜することが可能です」ではなく「〜できる」とする。
## ⑩ 視線の流れを意識する
左上から右、上から下へ自然に読める配置にする。  
重要情報は左上に置く。

## Behavior
【読み込みの場合】
 -from pptx import Presentationで読みこみ可能
 -文字コードを明示的に指定してテキストを再抽出
【書き込みの場合】
以下のステップで資料作成
- skills\slide\pptx_generator_simple2.py をtool readで呼び出して、ベースを理解する
- スライドのコンテンツを、スライドの条件に従って作成する
- TEMPLATES を確認する
- SLIDES_TO_USE でスライド構成を決める
- SLIDE_CONTENT でテキスト内容を編集
- pythoブロックのみ返却してtool writeで保存する（ファイル名は自由）
- 生成されたpythonコードをtool execで実行する

## Details
  STEP 1: SLIDES_TO_USE でスライド構成を決める
    使いたいテンプレート番号を配列で指定。順序が反映され、同じ番号の複数回使用OK。
    例: SLIDES_TO_USE = [1, 2, 3, 4, 5, 1]
        → 表紙(1) → 目次(2) → 棒グラフ(3) → 円グラフ(4) → テーブル(5) → 表紙(1)

  STEP 2: SLIDE_CONTENT でテキスト内容を編集
    キー = 出力順（1始まり）。SLIDES_TO_USE の順序に対応。
    例: SLIDE_CONTENT = {
            1: {"title": "表紙タイトル", "subtitle": "...", "credit": "..."},
            2: {"title": "目次", "items": ["項目1", "項目2", ...]},
            3: {"title": "売上推移", "categories": [...], "series": [...], ...},
            ...
        }
  STEP 3: 実行
    python pptx_generator.py
    → outputs/pptx_generator.pptx が生成される

【テンプレート一覧】全12種

  ■ 可変コンテンツ対応（items/categories/rows等の配列サイズに応じて自動調整）
    1: 表紙                    → title, subtitle, credit
    2: 目次（リスト）           → items[] ★可変: 1〜10項目程度OK
    3: 左グラフ・右テキスト     → categories[], series[], items[] ★可変
    4: 左テキスト・右グラフ     → categories[], series[], items[] ★可変
    5: 左テーブル・右テキスト   → columns[], rows[][] ★可変: 行列数自由
    6: 左テキスト・右グラフ     → categories[], series[], items[] ★可変
    7: カードグリッド（4枚）    → cards[] ★可変: 1〜4枚（4枚固定レイアウト）
    8: 左テキスト・右画像       → items[] ★可変, image_url

  ■ 固定レイアウト（項目数を変えるとレイアウト崩れの可能性あり）
    9:  3ポイント（丸囲み）     → items[3] ⚠固定: 3項目推奨
    10: ポイントリスト（コンパクト） → items[:4] ⚠上限: 最大4項目
    11: ポイントリスト（大）    → items[:3] ⚠上限: 最大3項目
    12: タイムライン           → items[:4] ⚠固定: 円+バー配置のため変更注意

【データ構造リファレンス】

  ● 箇条書き（items）
    "items": ["項目1", "項目2", "項目3"]
    → 配列の長さに応じて行数が変わる

  ● グラフ（chart）
    "categories": ["Q1", "Q2", "Q3", "Q4"],
    "series": [
        {"name": "2024年", "values": [100, 120, 140, 160]},
        {"name": "2025年", "values": [110, 130, 150, 170]}
    ],
    "chart_type": "COLUMN_CLUSTERED"  # or "PIE", "LINE"
    → categoriesの数、seriesの数は自由に増減可能

  ● テーブル（table）
    "columns": ["項目", "2023年", "2024年", "2025年"],
    "rows": [
        ["売上", "100億", "120億", "140億"],
        ["利益", "10億", "15億", "20億"]
    ]
    → 列数・行数は自由に増減可能
